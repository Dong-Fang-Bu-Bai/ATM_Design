# 交易处理模块设计文档

## 1. 模块概述

交易处理模块负责 ATM 系统的核心金融业务，包括取款、存款、转账三种交易类型，以及交易查询功能。本模块基于 Spring Boot 框架开发，采用事务控制确保数据一致性。

## 2. 交易实体设计

### 2.1 Transaction 实体字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 交易ID，主键自增 |
| transactionId | String | 对外返回的交易流水号，唯一标识 |
| accountId | Long | 关联账户ID |
| cardNo | String | 银行卡号 |
| transactionType | Integer | 交易类型：1-取款 2-存款 3-转账 |
| amount | BigDecimal | 交易金额 |
| balanceBefore | BigDecimal | 交易前余额 |
| balanceAfter | BigDecimal | 交易后余额 |
| transactionStatus | Integer | 交易状态：0-待处理 1-成功 2-失败 3-已撤销 |
| targetAccountNo | String | 目标账户（转账用） |
| targetBank | String | 目标银行（跨行转账用） |
| failureReason | String | 失败原因 |
| description | String | 交易描述 |
| createdAt | LocalDateTime | 创建时间 |
| completedAt | LocalDateTime | 完成时间 |

## 3. 交易状态

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 0 | PENDING | 待处理，交易刚创建，等待执行 |
| 1 | SUCCESS | 成功，交易已完成 |
| 2 | FAILED | 失败，交易执行失败 |
| 3 | CANCELLED | 已撤销，交易被撤销 |

## 4. 取款业务流程

### 4.1 业务流程

```
1. 用户输入取款金额
2. 系统校验取款金额（是否大于0，是否为100的倍数）
3. 系统校验账户余额是否充足
4. 系统校验单日取款限额（默认单日限额20000元）
5. 扣除账户余额，记录交易流水
6. 返回取款结果
```

### 4.2 关键业务规则

- 取款金额必须大于0
- 取款金额必须为100的整数倍
- 单次取款上限：5000元
- 单日累计取款限额：20000元
- 账户余额必须大于等于取款金额

## 5. 存款业务流程

### 5.1 业务流程

```
1. 用户输入存款金额
2. 系统校验存款金额（是否大于0）
3. 系统校验存款账户是否存在
4. 增加账户余额，记录交易流水
5. 返回存款结果
```

### 5.2 关键业务规则

- 存款金额必须大于0
- 单次存款上限：50000元
- 存款账户必须存在且状态正常

## 6. 转账业务流程

### 6.1 业务流程

```
1. 用户输入目标账户和转账金额
2. 系统校验转账金额（是否大于0）
3. 系统校验本账户余额是否充足
4. 系统校验目标账户是否存在
5. 系统校验目标账户不是本账户（不能给自己转账）
6. 扣除本账户余额，增加目标账户余额
7. 记录转出和转入两条交易流水
8. 返回转账结果
```

### 6.2 关键业务规则

- 转账金额必须大于0
- 单次转账上限：10000元
- 单日转账限额：50000元
- 本账户余额必须大于等于转账金额
- 目标账户必须存在
- 目标账户不能与本账户相同

## 7. 接口设计

### 7.1 取款接口

- **路径**: `POST /api/atm/transactions/withdraw`
- **请求参数**: sessionId, amount, printReceipt
- **响应**: 交易结果、交易流水号、剩余余额

### 7.2 存款接口

- **路径**: `POST /api/atm/transactions/deposit`
- **请求参数**: sessionId, amount, printReceipt
- **响应**: 交易结果、交易流水号、更新后余额

### 7.3 转账接口

- **路径**: `POST /api/atm/transactions/transfer`
- **请求参数**: sessionId, targetAccountNo, amount, printReceipt
- **响应**: 交易结果、交易流水号、剩余余额

### 7.4 交易查询接口

- **路径**: `GET /api/atm/transactions/{transactionId}`
- **请求参数**: transactionId
- **响应**: 交易详细信息

## 8. 异常分支处理

| 异常情况 | 处理方式 |
|----------|----------|
| 余额不足 | 返回失败，transactionStatus=2，failureReason="余额不足" |
| 超取款限额 | 返回失败，transactionStatus=2，failureReason="超单次取款限额" |
| 超单日限额 | 返回失败，transactionStatus=2，failureReason="超单日取款限额" |
| 无效金额 | 返回失败，transactionStatus=2，failureReason="无效金额" |
| 目标账户不存在 | 返回失败，transactionStatus=2，failureReason="目标账户不存在" |
| 自转账 | 返回失败，transactionStatus=2，failureReason="不能向本人账户转账" |
| 系统错误 | 返回失败，transactionStatus=2，failureReason="系统错误" |
