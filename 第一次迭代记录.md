## 2. 当前完成范围

### 前端模块

- 已建立 Vue 3 + Vite 前端工程。
- 已实现欢迎页、登录页、主菜单页、余额查询页。
- 已预留取款、存款、转账、修改密码、交易凭条等后续入口。
- 已通过 Pinia 保存 ATM 会话状态。
- 已通过 Axios 封装统一接口调用。

### 认证与账户模块

- 已建立 Spring Boot 后端工程 `atm-server-auth`。
- 已实现登录、退出登录、账户信息查询、余额查询接口。
- 已提供 `dev` profile，可使用 H2 内存数据库自动建表并初始化演示账号。
- 已提供本地一键启动脚本 `start-dev.cmd`。

### 交易模块

- 交易模块保持独立目录 `transaction/`。
- 已提供交易模块设计文档、交易实体、DTO、Controller、Service 骨架。
- 主后端通过 Maven `build-helper-maven-plugin` 引入 `transaction/src/main/java`，因此交易模块保持独立目录的同时，运行时可被主 Spring Boot 应用扫描和挂载。
- 第一次迭代阶段交易接口已接入，但完整业务逻辑尚未实现；当前返回 `501 Not Implemented`，说明完整取款、存款、转账逻辑将在第二次迭代完成。

## 3. 字段与接口口径统一

本次整理后，对外接口统一采用以下口径：

| 类别 | 统一口径 |
| --- | --- |
| 会话字段 | `sessionId` |
| 账户信息接口 | `GET /api/atm/accounts/profile?sessionId=...` |
| 余额查询接口 | `GET /api/atm/accounts/balance?sessionId=...` |
| 退出登录请求 | `{ "sessionId": "..." }` |
| 交易请求身份字段 | 统一传 `sessionId`，不要求前端传 `cardNo` |
| 交易流水对外字段 | `transactionId` |

后端 `AccountController` 仍兼容旧路径和旧参数名，便于过渡；但前端、OpenAPI、接口测试文件均已按上述统一口径整理。

## 5. 验收链路

第一次迭代核心验收链路：

1. 打开前端欢迎页。
2. 进入登录页。
3. 输入演示卡号和密码。
4. 登录成功后进入主菜单。
5. 点击查询余额。
6. 后端返回余额数据并在前端展示。

交易模块验收方式：

1. 访问 `POST /api/atm/transactions/withdraw`、`deposit` 或 `transfer`。
2. 后端返回 `501`。
3. 响应信息表明交易模块已接入后端，但当前为第一次迭代骨架。

该结果符合第一次迭代中“交易模块设计文档、实体类和服务类骨架”的交付范围。
