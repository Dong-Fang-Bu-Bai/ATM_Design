## 4. 推荐执行顺序

建议按下面顺序推进，这样阻塞最少：

1. 先补数据库脚本和演示数据，否则后端真实联调无法稳定复现。
2. 再补交易模块设计文档和代码骨架，完成同学 C 在第一次迭代中的最低交付要求。
3. 然后完成一次真实联调，确认登录和余额查询不是只停留在代码层面。
4. 最后统一接口文档和验收说明，把第一次迭代材料整理成可提交、可展示、可复现的状态。

- 修改密码完整实现
